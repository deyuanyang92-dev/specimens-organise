# 标本入库管理软件

本项目实现一个本地桌面版标本入库工作台（Windows / Linux），基于 PyQt5，数据保存在当前工作区的 `数据/` 目录中。

## 启动

```bash
python run_app.py
```

如果要指定工作区：

```bash
python run_app.py --workspace "D:\your\workspace"
```

依赖：

```bash
pip install -r requirements.txt
```

依赖说明：`PyQt5` 为 GUI 框架，`openpyxl` 处理 Excel，`Pillow` 处理缩略图，`tifffile` 支持 TIFF 大图。

## 已实现功能

- 新增标本自动生成 `YZZ000001` 格式入库编号。
- 当前工作区内编号强制唯一，启动、新建、导入前会校验重复。
- 标本信息、照片信息、分类信息分别写入独立 Excel。
- 左侧编号列表显示三类状态：标本信息、照片信息、分类信息。
- 字段自动保存，并追加写入 `修改记录.xlsx`。
- `管内编号*` 自动解析采集日期和采集地点缩写。
- 分类信息实时读取 `字段模版/表格信息预设字段.xlsx`，输入种名片段后可选择候选并自动填充。
- 照片支持三种管理方式：复制到工作区照片库并记录绝对路径、仅记录绝对路径、复制到自定义照片库并记录绝对路径；复制时保留原图文件，不压缩、不移动源文件。
- 照片批量关联会一次性写入 Excel，并在后台生成缩略图缓存，避免大 TIFF 文件阻塞界面。
- 启动时先显示主窗口，再延迟加载编号列表、首张照片预览和图片索引，减少打开工作区时的卡顿。
- 支持通过“新窗口”同时打开不同工作区；同一工作区重复打开会聚焦已有窗口，避免多个编辑窗口同时写同一批 Excel。
- Windows 构建使用快速启动目录版（`--onedir`），减少单文件 EXE 每次解包造成的启动卡顿。
- “检索图片”会扫描当前工作区图片，按当前标本的入库编号、管内编号、地点缩写、物种和科信息排序；支持 Ctrl/Shift 多选后批量关联到当前标本。
- 图片检索框默认使用 `管内编号*` 派生的瓶内标签，例如 `QD-CK-SC008-260827` 会按 `QD-CK-SC008` 检索。
- 右侧“照片信息”表格会显示当前标本的全部照片记录，包括序号、文件名、相对路径和描述；下方照片信息栏同时显示绝对路径；点击任意行会切换预览。
- 照片预览支持单张和 2/4/6/8 宫格显示；宫格中点击照片可放大为单图，再次点击单图返回宫格。
- 照片预览区支持右键菜单，可放大/返回宫格、打开原图、打开原图所在位置、删除照片、复制相对路径或绝对路径。
- 图片预览支持 `Ctrl+鼠标滚轮` 缩放、滚轮切图、鼠标拖拽平移、双击适配，并可用系统默认图片查看器打开原图。
- 启动时优先打开上次使用的工作区；界面显示当前工作目录，并支持切换/初始化工作区。
- 软件不会在 exe 所在目录自动创建数据文件；选择未初始化目录时，会先弹窗请求许可。
- 撤回/返回覆盖字段、照片、新建、删除等数据操作，默认保存 200 步，可在设置中调整。
- 导入旧工作区前预检编号冲突；冲突时阻止导入并生成 `导入冲突报告_*.xlsx`。
- 版本管理窗口支持创建/回退工作区数据快照，并列出 `releases/` 下的历史软件版本。

## 数据文件

正式数据不会写回模板文件，而是写入：

- `数据/标本信息.xlsx`
- `数据/照片信息.xlsx`
- `数据/分类信息.xlsx`
- `数据/编号索引.xlsx`
- `数据/修改记录.xlsx`
- `数据/操作记录.xlsx`
- `数据/数据版本记录.xlsx`
- `数据/工作区配置.json`
- `数据/数据版本/`

默认会把外部照片原样复制到工作区 `照片/` 并记录绝对路径，源文件保留不变。也可以在设置中改为仅记录绝对路径，或复制到自定义照片库目录。

## 构建 Windows 版本

**一键打包**：双击 `build.bat`，自动安装依赖并构建 EXE。

**手动打包**：

```bash
pip install pyinstaller
python build_release.py
```

构建后会生成：

- `dist/标本入库管理/`：最新稳定版目录
- `releases/v0.2.4/`：带版本号的历史版本 + 构建信息 + SHA256 校验

详细教程见 [docs/build-windows.md](docs/build-windows.md)。
