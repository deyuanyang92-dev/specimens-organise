# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

标本入库管理 (Specimen Inventory Management) — a PyQt5 desktop app for managing biological specimen records stored as Excel files. Target platforms: Windows and Linux.

## Commands

```bash
# Install dependencies
pip install -r requirements.txt

# Run the app (auto-discovers last-used workspace or current directory)
python run_app.py
python run_app.py --workspace "/path/to/workspace"

# Run full test suite
python -m unittest discover -s tests

# Run a single test
python -m unittest tests.test_core.CoreTests.test_create_vouchers_increment

# Build Windows/Linux release (requires PyInstaller)
python build_release.py --version 0.5.0

# Windows: one-click build (also runs build_release.py)
build.bat
```

## Architecture

Entry point: `run_app.py` → `specimen_app/main.py:main()` → `specimen_app/ui.py:run_app()`.

### Core modules

- **`models.py`** — All constants: Excel file names, headers (imported from `classification_fields.py`), required fields, save method options, category mappings, `CURRENT_DATA_SCHEMA_VERSION`. Also the inventory-summary view constants: `SUMMARY_COLUMNS` (flat wide-table column order = specimen fields + classification fields + `照片数` + photo-aggregate columns `照片文件名`/`照片绝对路径`/`照片描述` = `PHOTO_AGGREGATE_COLUMNS`; classification `备注` disambiguated as `分类备注` via `CLASSIFICATION_NOTE_DISPLAY`), `SUMMARY_COLUMN_SOURCE` (display col → `(category, excel_field)`, `category="readonly"` marks 入库编号*/照片数/photo-aggregate cols), `SUMMARY_DEFAULT_VISIBLE_COLUMNS` (photo-aggregate cols are NOT in it — hidden by default). Dataclasses: `StatusFlags`, `ImportResult`, `ActionResult`, `Row` alias. Custom exceptions: `WorkspaceError`, `WorkspaceLockedError`, `WorkspaceNotInitializedError`, `DuplicateVoucherError`, `ImportConflictError`.
- **`classification_fields.py`** — Classification column definitions and autofill mappings. `CLASSIFICATION_COLUMNS` defines Excel write order; `SPECIES_LOOKUP_INPUT_COLUMNS` / `FAMILY_LOOKUP_INPUT_COLUMNS` dictate which fields trigger taxonomy preset searches. `classification_values_from_species_match()` and `classification_values_from_family_match()` convert `SpeciesMatch`/`FamilyMatch` results into classification form values (e.g., `SpeciesMatch` → `{种名*: "...", 种拉丁: "...", 属名: "...", 科*: "...", 科拉丁: "..."}`).
- **多人协作（M1-M5，v0.5.0+）**：详见 `docs/multi-user-collaboration.md`。`ExcelStore.ensure_assignee_series(assignee, prefix)` 给指定录入员创建/复用独立编号系列（前缀分人，避免离线撞号）。`ExcelStore.import_workspace(source, photo_duplicate_policy)` 加 `photo_duplicate_policy` 参数：`"import"`（默认，向后兼容现有手动「导入工作区」）/ `"skip"` / `"report"`（跨 voucher 同 SHA256 → `duplicate_candidates`）。`ExcelStore.detect_legacy_workspace()` / `upgrade_to_multi_user_protocol()` 让旧工作区一键升级（只加 `multi_user_protocol_version` / `legacy_yzz_segment` 两个配置键，强制 snapshot 可回退，零改 Excel 数据）。`BatchGenerateDialog` 加 "录入员独立前缀" + "同时生成任务包 zip" 两个勾选项。
- **`excel_store.py`** — `ExcelStore` is the data layer backing the entire app. Reads/writes all Excel files via openpyxl. Handles CRUD, undo/redo (action-log based, default 200 steps), workspace import with fingerprint-based conflict detection, data snapshots (`create_data_snapshot` / `restore_data_snapshot`), and file locking (`数据/.workspace.lock`). Blocks numpy import during openpyxl usage to prevent library conflicts. Uses in-memory cache (`_row_cache`) to avoid re-parsing files on every read. `summary_records()` left-joins specimen + classification + photo count into one flat dict per voucher (keyed by `SUMMARY_COLUMNS`), plus per-voucher photo-aggregate **list** values for `照片文件名`/`照片绝对路径`/`照片描述` (all photos of that voucher) — a pure in-memory view backing the inventory-summary wide table, changes nothing on disk. Photos from outside the workspace are copied into `照片/` with content-hash deduplication; same-name photos get numbered archive names (`same.jpg` → `same_2.jpg`); hash-prefixed legacy archives are migrated on open. `downgrade_schema_version(target="1.0.0")` writes `target` into `工作区配置.json`'s `data_schema_version` key, allowing an older software version to open a workspace that was upgraded by a newer version (data content unchanged — only the version flag is lowered). `batch_reserve_vouchers(n, series_name)`/`log_alloc_event(record)`/`read_alloc_log()` — 编号分发审计日志（`编号分发记录.xlsx`）：批量预留编号段、追加分发日志行（批量领取/任务开始/任务结束）、读取日志；详见 `docs/voucher-number-document.md`。
- **`ui.py`** — `run_app()` creates the `QApplication` and `SpecimenApp(QMainWindow)`. Main window (~1500+ lines): voucher list with status flags (√/× for specimen/photo/classification), photo preview with zoom/pan/grid (2/4/6/8), species autocomplete, image search dialog, version manager, settings. Background photo loading via `QThread`. 左侧面板顶部有**录入任务门控**：`_active_task` dict 状态 + `_start_task()`/`_end_task()`/`_update_task_indicator()`，「新增入库编号」默认 disabled，必须先「开始录入任务」才能启用（硬门控）；`_StartTaskDialog`（录入人/用途/备注）、`BatchGenerateDialog`（批量预留编号段 + 导出 xlsx/csv）、`WorkloadReportDialog`（录入人员汇总+明细+导出，含数据文件路径提示）分别由「开始录入任务」/「编号→批量生成编号」/「工具→入库人员记录」触发。A persistent yellow `QLabel` banner (`_preset_warning_banner`) is inserted below the workspace toolbar on startup and shown (never auto-hidden) only when the taxonomy preset file is missing — visible signal that species/family autofill is inactive. The 工具 menu includes `"降低工作区兼容版本…"` (`_downgrade_workspace_schema`): password-gated, offers an optional snapshot, then calls `store.downgrade_schema_version("1.0.0")` so an older software version can re-open the workspace. `IngestSummaryDialog` (toolbar "入库汇总") is a **non-modal, single-instance, read-only** wide-table view over `store.summary_records()`: every voucher's specimen + classification fields + `照片数` + photo-aggregate columns (`照片文件名`/`照片绝对路径`/`照片描述`, list values rendered `"; "`-joined with a full tooltip via `_summary_cell_text`, hidden by default) in one sortable, per-column-filterable table (left-click header sorts; right-click **anywhere — header or body** opens the filter menu via `_voucher_column_menu`). Column visibility is managed by a closable side panel (`_build_column_panel`, hidden by default, toggled by the nav-bar "列设置" button), persisted to `summary_visible_columns`. Plus search, pagination, a "刷新" button, and Excel/CSV export of the current filtered+visible columns. The table is read-only — **double-clicking a row jumps to the main window** via `SpecimenWindow.reveal_voucher` (clears the main window's search/filter if needed, navigates to the voucher's page, selects+scrolls to the row, which loads it into the editor) and focuses the main window; the dialog stays open. `open_ingest_summary` keeps one instance (`SpecimenWindow._ingest_summary_dialog`), focusing+refreshing it instead of opening a duplicate. Photo management (replace / delete-unlink / move / open-original) lives in the **main window's photo-table right-click menu**, not in this dialog. The summary table is **`ExtendedSelection`** (Windows-style Ctrl/Shift multi-select); body right-click adds a "导出选中照片" action (`_selected_vouchers` → `_export_selected_photos` → `BatchExportDialog(..., photo_focus=True)`, exports photos *of the selected vouchers*, not selected photo files) above the per-column filter items. The search bar has an "导入编号列表" button (`_open_voucher_list_filter`): paste or load a voucher list from txt/csv/xlsx → `_voucher_list_filter` set, applied in `_apply_filters` alongside global search + column filters; "清除所有筛选" clears it too. `ImageSearchDialog` result cards carry full-name tooltips and a "查看详情" context action since card labels elide long filenames.
- **`parsing.py`** — Voucher number formatting (`YZZ000001` pattern), tube number parsing (extracts location codes, dates, bottle labels from strings like `QD-LSD-SC001-1-R-250923`). Also exports `extract_tube_from_filename()` and `extract_photo_date()` for auto-filling specimen info from photo filenames via `_TUBE_PATTERN` and `_PHOTO_DATE_RE` regexes.
- **`workspace.py`** — Workspace discovery (`default_workspace()` checks last-used, CWD, executable parent), validation (rejects `build/`, `dist/`, `releases/` directories via `is_generated_workspace_path`; rejects filesystem/drive root and home dir via `is_unsafe_workspace_root` — those are too large and would make full-workspace scans walk huge trees), initialization (copies `字段模版/` from template).
- **`species.py`** — `SpeciesMatcher` loads species presets from `字段模版/表格信息预设字段.xlsx` (columns: 物种中文名, 物种拉丁名, 科中文名, 科拉丁名). Provides fuzzy matching against Chinese and Latin names (ranking: exact > prefix > contains). Auto-reloads when file mtime changes. Returns `SpeciesMatch` (chinese_name, latin_name, family_name, family_latin; `genus_name` derived from the first word of latin_name) and `FamilyMatch` (family_name, family_latin) frozen dataclasses. These feed into `classification_fields.py` converters for autofill.
- **`image_cache.py`** — `ThumbnailCache` stores JPEG thumbnails in `数据/缩略图缓存/`, keyed by source path + file size + mtime + target size. TIFF handling via tifffile/numpy (lazy-imported, uses stride downsampling for large files). In-memory LRU cache (default 64MB). `load_source_image` enforces a `_MAX_DECODE_PIXELS` (~24MP) decode cap — JPEGs use `Image.draft()` and any over-cap image is `reduce()`-downsampled *before* exif/convert/thumbnail allocate full-resolution buffers, so gigapixel specimen scans can't exhaust memory. TIFF pages get a forced stride cap even when no target size is given.
- **`startup_diag.py`** — `mark(stage)` logs per-stage elapsed time + peak process RSS to stderr (`[startup]` prefix) and appends to a size-capped `startup_diagnostics.log` in the app config dir (not the workspace). Hooked into `ExcelStore.__init__` substeps and `SpecimenWindow` startup so a freeze/hang can be pinpointed to a stage. Pure stdlib; logging failures never block startup.
- **`image_search.py`** — Scans workspace for images (excluding build/dist/releases/data dirs), builds disk-cached index. Scores images against core identifiers (extracted from tube numbers as `XXX-XXX-NNNNNN`). Supports query filtering, linked-photo detection, index appending without rescan.
- **`app_settings.py`** — Persists last workspace, recent workspaces, preview quality, search paths, grid filename toggle, window geometry, update-check preferences (`check_updates_on_startup`, `last_update_check`), inventory-summary visible columns (`summary_visible_columns`; empty = fall back to `SUMMARY_DEFAULT_VISIBLE_COLUMNS`), global UI font size (`ui_font_size`; `0` = use system default, otherwise clamped to 7–24 pt), custom image-viewer path (`image_viewer_path`; empty = OS default) in `%APPDATA%/标本入库管理/settings.json` (Windows) or `~/.specimen_inventory/settings.json` (Linux).
- **`release_manager.py`** — `list_releases()` discovers versioned release directories under `releases/`, identifies executable files by platform-specific rules.
- **`worms_client.py`** — WoRMS REST API 客户端（stdlib urllib）+ 本地 SQLite 缓存（`~/.specimen_inventory/worms_cache.sqlite`）。`query_worms()` 按 `socket.gaierror`/`timeout`/`ssl.SSLError` 分类报错，给出中文可操作提示。`ensure_bootstrap_cache()` 在 `run_app()` 启动时检查：若用户缓存为空且 `specimen_app/assets/worms_cache_bootstrap.sqlite.gz` 存在（PyInstaller `--add-data` 打入），自动调 `install_cache_gz()` 解压安装，实现离线首启开箱即可查 WoRMS 分类。bootstrap 文件由 `tools/build_worms_cache.py --download` 在联网机器季度生成。详见 `docs/Worms-update-20260516.md`。
- **`crash_log.py`** — E1 崩溃日志 + 启动检测。`install_excepthook()` 装 `sys.excepthook` + `threading.excepthook`（Py3.8+）；任何未捕异常 → 写 `~/.specimen_inventory/crash_<ts>.log`（含堆栈 + 平台 + 版本，UTF-8）。`mark_app_started()` 启动时检测上次是否 clean exit；`mark_app_exiting_clean()` closeEvent 末尾调。`list_recent_crash_logs()` 供 UI 列表展示。保留最近 20 个 crash log，更老自动清理。`ui.run_app()` 第一时间装 hook；启动后 ~2s 异步弹 QMessageBox 提示"上次异常退出"（如有）。
- **`worms_crawler_daemon.py`** — W1 WoRMS 抓取独立进程 daemon。`spawn_detached(state, pid_file)` 用 `subprocess.Popen` 跨平台 detach（POSIX `start_new_session=True` / Windows `DETACHED_PROCESS | CREATE_NEW_PROCESS_GROUP`）。`daemon_running(pid_file)` 检测进程存活（`os.kill(pid, 0)`）。`request_stop_daemon(pid_file)` 发 SIGTERM (POSIX) / `taskkill` (Windows) 优雅停止；state 已 autosave 可断点续传。`run_daemon()` daemon 主流程含 PID 文件管理 + 信号处理 + 兜底 crash log。UI 入口：「管理本地 WoRMS 数据库」对话框「在独立后台进程下载」勾选项 + 「检查后台下载状态」「停止后台下载」按钮。
- **多端关闭兼容（E3/C1/C2）**：`SpecimenWindow.closeEvent` 的 `_stop_worker` helper 把所有 QThread 的 `wait(timeout)` 超时后强杀（`terminate()`）；`WindowManager.register_dialog_stopper` / `stop_all_dialog_workers` 让主窗口接管所有打开对话框的后台 worker；`ExcelStore.__init__` atexit 钩子改 `_release_lock_with_timeout(3.0)` 用守护线程限时释放，防 SMB/NAS 网络锁挂死。
- **`dwc_export.py`** — A1 Darwin Core Archive (DwC-A) 导出。`export_dwc_archive(store, dest_zip, dataset_title=, dataset_creator=)` 把工作区一键转 zip（含 `meta.xml` + `occurrence.txt` + `multimedia.txt` + `eml.xml`）。中文列 → DwC term 映射：`入库编号*` → occurrenceID+catalogNumber、`管内编号*` → fieldNumber、`种拉丁/科拉丁` → scientificName/family、`采集日期` → eventDate、`采集地点缩写*` → verbatimLocality、`信息录入人员/核对人员` → recordedBy/identifiedBy 等。照片走 Audubon Core Multimedia 扩展（identifier/accessURI/fileFormat/hashFunction/hashValue）。**纯只读**导出，不改工作区。UI 入口：工具菜单「导出 Darwin Core Archive…」。生成的 zip 可用 GBIF IPT / iDigBio Validator 校验、上传到 GBIF 节点。
- **`exif_info.py`** — A2 EXIF 元数据抽取与回填。`extract_exif(path)` 用 lazy import PIL 抽 DateTimeOriginal/GPSInfo/Make/Model（无 PIL / 无 EXIF / 损坏照片都 graceful 返回 `{}`）。`apply_exif_to_specimen(store, voucher, photo_path, overwrite=False)` 默认只填空字段不覆盖已有值。UI 入口：工具菜单「从 EXIF 批量回填采集日期…」— 扫所有"有照片但采集日期为空"的 voucher，从第一张照片读 EXIF 回填。
- **`server_sync.py`** — M1 多人协作收件箱聚合 + S1/S2/S3/S7 增强。**S1 嵌套扫描**：`_discover_candidates` 递归找含 `数据/` 子目录的工作区，最多 3 层深；子目录名扁平化（`__` 拼）防同名撞锁。**S2 多源直接合并**：`aggregate_sources(store, source_dirs)` 接受任意源目录列表，不需 incoming 中转，源目录原样保留。**S3 同名照片冲突报告**：聚合后扫所有 processed 子目录的 `照片信息.xlsx`，跨子目录"原始文件名相同但 SHA256 不同"的项写到 `incoming/name_conflicts/同名照片冲突报告_*.xlsx`。**S7 dry-run 预览**：`preview_aggregate(store, incoming_root) -> AggregatePreview` 只读扫描预测新增/重复/冲突/同名冲突/跨 voucher SHA256 数，UI 在「从收件箱聚合」按钮上集成预览 → 用户确认后才真合并。`aggregate_incoming(store, incoming_root, progress_cb)` 扫 `incoming/` 下所有"含 `数据/` 子目录"的文件夹，强制创合并前快照，逐个加锁（rename → `processing_*`）后调 `store.import_workspace(子目录, photo_duplicate_policy="report")`，按结果分流到 `processed/` / `conflicts/` / `errors/` / `duplicates/`。**降级模式**：无 `manifest.json` 也能合并（等价 P1 极简方案）。M4 段守护：有 manifest 时校验源 voucher 必须 ⊆ `manifest.voucher_range`，超界 → `errors/`。M4 跨 voucher 同 SHA256 照片审核：result.duplicate_candidates 写到 `duplicates/{subdir}_重复审核_*.xlsx`，对应 photo 行**不写入中心**，由主管人工决定。UI 入口：工具菜单 → "从收件箱聚合…"（密码门控复用 `ADMIN_PASSWORD`）。详见 `docs/multi-user-collaboration.md`。
- **`task_package.py`** — M3 任务包导出/导入。`export_task_package(store, assignee, count, prefix, dest_zip)` 在中心机：调 `store.ensure_assignee_series` 取/建录入员独立系列（前缀分人）→ `batch_reserve_vouchers` 预留段 → 写 `alloc_log`「任务开始」→ 用 `ExcelStore` 在临时目录建空工作区骨架（含该系列定义 + 字段模版/）→ 写 `manifest.json` → 打 zip。`import_task_package(zip, target)` 在录入员侧：zip-slip-safe 解压到空目录，验证 manifest + 工作区配置存在。`read_task_manifest(zip)` 不解压窥探。UI 入口：编号→批量生成编号 Dialog 加 "同时生成任务包 zip" 勾选项。
- **`updater.py`** — In-app GitHub update support (stdlib only, no new deps). `check_latest_release()` queries the repo's latest GitHub Release, picks the platform zip + `update_manifest_{plat}.json` assets; `is_newer()` / `_parse_version()` compare versions (prerelease suffixes like `-test.1` sort below the matching release). `download_update()` is the UI entry point: with a manifest it does **incremental update** — splits into a small `app_*.zip` and a content-hash-named `runtime_*.zip`, and if a local installed release has a matching `runtime_hash` (read from each bundle's `.update_meta.json`), it reuses that runtime locally and downloads only the app zip; otherwise downloads both; no manifest → falls back to `download_release()` (full zip). All downloads enforce sha256 verification and zip-slip-safe extraction into a temp dir before moving to `releases/v{version}/`. Only HTTPS + `github.com`/`githubusercontent.com` hosts allowed. Does not auto-launch — version switching stays manual via `VersionManagerDialog`.
- **`icon.py`** — App icon. `create_app_icon()` still draws the classic icon programmatically. `APP_ICON_VARIANTS` lists 4 pre-generated variants (`specimen_blue`/`ledger_green`/`photo_coral`/`archive_indigo`) shipped under `assets/icons/app-icon-variants/` (built by `tools/generate_icon_variants.py`); `get_app_icon(variant)` loads the chosen variant's `.ico` (resolved across source / PyInstaller `_MEIPASS` / cwd roots) and falls back to the programmatic icon if missing. `DEFAULT_APP_ICON_VARIANT = "specimen_blue"`.
- **`field_help.py`** — Access layer for the bundled `字段模版/` reference data: `bundled_template_path(filename)` (multi-root resolver, source / `_MEIPASS` / `_internal/`), `load_field_help()` (parses `数据录入字段及字段说明.xlsx` Sheet2 → `{字段名: {示例,说明,其他要求}}`, cached, fault-tolerant), `field_help_for(panel_field)` (maps photo-panel field names via aliases). openpyxl-only, import-safe.
- **`cursors.py`** — Fun cartoon cursors that replace the default arrow, picked in Settings. `CURSOR_STYLE_OPTIONS` (key→显示名: default/finger/palm/pen/paw/star), `create_cursor_image()` draws each procedurally via `QImage`+`QPainter` (no image assets, import-safe — no `QApplication` needed), `cursor_hotspot()`, `make_cursor()` builds the `QCursor` (lazy — needs a GUI). Persisted as `cursor_style` in settings.json.
- **`theme.py`** — Shared light UI theme: semantic palette constants (consolidates the formerly scattered hex like `#2a6fbd`) + `APP_QSS` global stylesheet + `apply_app_theme(app)`, applied once in `run_app()` after font setup. Styles menus/buttons/inputs/table headers/toolbar/groupbox/tabs/scrollbars for a consistent Geneious/PhyloSuite-like look. **The QSS deliberately sets no `font`/`font-size`** — fonts stay owned by `apply_app_font_size()` + `_refresh_scaled_fonts()`. Existing per-widget `setStyleSheet` calls still win (widget-level QSS overrides app-level).
- **`batch_export.py`** — `BatchExportDialog` for exporting specimen data + photos by voucher number list (like NCBI Batch Entrez). Parses voucher numbers from pasted text, writes a multi-sheet Excel workbook, optionally copies photo files and packages as ZIP. Accessible from right-click menu ("批量导出选中") or toolbar button. Photo export supports a format/compression group (`PHOTO_EXPORT_FORMATS`: "保持原格式"=`shutil.copy2` as before, else Pillow re-encode to JPG/PNG/TIFF via `_reencode_photo` — JPEG quality slider, optional max-edge downscale); the `photo_focus=True` ctor flag (used by the summary dialog's "导出选中照片") makes it a pure photo exporter — photo-files-only defaults, retitled dialog, and the 标本信息/分类信息/照片路径清单 checkboxes hidden (still constructed so `_do_export` is unchanged).
- **`__init__.py`** — Only exports `__version__` (currently `"0.5.0"`), used by `build_release.py` as the default build version.

### Data storage

All user data lives in the workspace's `数据/` directory as `.xlsx` and `.json` files:
- `标本信息.xlsx`, `照片信息.xlsx`, `分类信息.xlsx` — main data tables
- `编号索引.xlsx` — voucher index with UUIDs and record fingerprints
- `修改记录.xlsx` — two-sheet workbook (修改明细 + 修改汇总)
- `操作记录.xlsx` — undo/redo action log
- `数据版本记录.xlsx` — snapshot version history
- `工作区配置.json` — workspace config (prefix, next serial, undo depth)
- `数据版本/` — snapshot file copies
- `.workspace.lock` — process-level lock file
- `编号分发记录.xlsx` — 分发审计流水（批量领取/任务开始/任务结束三类记录），老工作区打开时自动创建

Templates live in `字段模版/`. External photos are archived (copied) into `照片/` with content-hash deduplication; same-name photos get numbered suffixes (`same.jpg` → `same_2.jpg`). Internal photos already in the workspace are left in place with relative paths recorded. The workspace is fully portable.

## Key conventions

- **Data compatibility is mandatory.** Every update must remain compatible with data that users have already entered. Do not break existing Excel/JSON files, field names, voucher numbers, UUIDs, relative photo paths, fingerprints, version snapshots, or undo/redo logs without a safe backward-compatible reader, migration path, or rollback plan. Before changing schemas, save/import/export behavior, or generated file formats, verify that previously recorded workspace data still opens and remains usable.
- **Preserve old logic with comments when updating.** For future code updates in this project, do not silently delete important original logic. Prefer making changes with `#` comments that record the old behavior, why it changed, and how compatibility is preserved. If code must be removed because keeping it would be unsafe or misleading, leave a concise `#` comment near the replacement explaining the original path and the compatibility decision.
- Excel files are fully rewritten on every save (`_write_plain_rows` creates a fresh `Workbook` each time), so openpyxl's read_only mode is used for all reads.
- Undo/redo is action-log based: every mutation appends to `操作记录.xlsx` with old/new JSON snapshots; undo applies the inverse operation.
- The `_loading` flag in UI prevents save handlers from firing during programmatic field updates.
- Field edits auto-save via `schedule_save` (500ms debounce). The toolbar "自动保存" checkable action toggles `SpecimenWindow.auto_save_enabled` (persisted as `auto_save_enabled` in settings.json, default True): when off, `schedule_save` registers the timer in `_save_timers` but does **not** start it — so the manual "保存" buttons' `_flush_pending_saves` still writes pending edits.
- All user-facing text is Chinese (labels, field names, file names). Code identifiers are English.
- When `管内编号*` is updated, `采集日期`, `采集地点缩写*`, and `保存方式` are auto-derived from the tube number.
- `字段模版/` (taxonomy preset `表格信息预设字段.xlsx` + per-field help `数据录入字段及字段说明.xlsx`) is **bundled with the software** — `specimen_app/字段模版/` is committed and `--add-data`-bundled by `build_release.py`. `field_help.py:bundled_template_path()` resolves it across source / PyInstaller `_MEIPASS` / `_internal/` roots. `SpeciesMatcher` is built via `ui._species_matcher()` from the bundled preset (NOT the workspace) — so species/family auto-match works in any workspace regardless of whether it has `字段模版/`. `SpecimenWindow._wrap_field_with_hint(field, widget)` wraps each entry control so fields with help text in `数据录入字段及字段说明.xlsx` (Sheet2: 大类/字段名称/填写示例/字段说明/其他要求) get a small low-key "?" `QToolButton` **after** the input box (grey, hover = summary tooltip + turns blue, click = full `QMessageBox`); fields with no help text are returned unwrapped. `initialize_workspace` still copies `字段模版/` into brand-new workspaces (legacy, harmless — the matcher no longer reads it).
- Data-entry speedups for the fields in `CARRY_OVER_SPECIMEN_FIELDS` (`models.py`: 标本存放位置/信息录入人员/核对人员/保存方式/备注): `new_specimen()` **always** carries these over from the current record — carry-over is a fixed workflow rule, not a toggle (the old toolbar "沿用上条信息" checkbox was removed; the `carry_over_specimen_fields` settings field is kept only for backward-compatible settings.json reads and no longer has any UI). The "＋新增入库编号" button (formerly "＋新增标本" in the toolbar) now lives at the top of the left voucher panel. The voucher list's right-click "批量设置标本信息" opens `BatchSpecimenFieldsDialog` to write them to all selected vouchers via `store.set_fields` (one undoable action-log entry per voucher).
- Workspace lock detects stale locks (PID not running or lock older than 10 minutes).
- The admin password is the module-level `ADMIN_PASSWORD` constant in `ui.py` (value `"123"`, unchanged) — used by voucher delete/batch-delete and by the 工具 menu's "用 Excel 打开数据文件…" (`_open_data_in_excel`). That action gates on the password, then warns (the app does not watch for external changes and rewrites whole files on save, so external edits can be clobbered), offers a `create_data_snapshot` first, opens the `数据/` dir via `_open_path`, and tells the user to reopen the workspace afterward.
- Global UI font size is adjustable: the settings dialog "界面字体大小" spinbox and the Ctrl+加/减/0 shortcuts (`_zoom_font`/`_zoom_font_reset`) call the module-level `apply_app_font_size()` (`QApplication.setFont`) and persist `ui_font_size`. `_default_app_font_point` (captured in `run_app`) is the reset target. Only "body" fonts scale — `voucher_table`'s Consolas font and its column widths follow via `_refresh_scaled_fonts()` (base values in `SpecimenWindow._VOUCHER_COL_BASE_WIDTHS` / `_VOUCHER_TABLE_BASE_PT`); the ~8 hard-coded stylesheet `font-size` declarations (panel titles, dashboard, filter buttons, etc.) intentionally do NOT scale.
- Image search defaults to TIFF files; users can switch to JPG-only or combined TIFF+JPG.
- Main-window photo preview: double-clicking a grid cell expands it to single view; in that grid-expanded state, double-clicking the image again returns to the original grid (`PhotoGraphicsView.return_requested` signal + `set_double_click_returns`; the "返回网格" button still works). Outside that state, double-click toggles fit↔100%. "打开原图" goes through the module-level `open_image_external()` — uses the settings `image_viewer_path` if set and valid, else falls back to `_open_path()` (OS default); the original is never loaded full-res into the in-app preview.
- Photo "删除" is always labelled **"取消关联"** in the UI (control-bar button, all three right-click menus, confirmation dialogs): `delete_photo`/`delete_photo_at`/`_delete_selected_photos` (method names unchanged) only remove the `照片信息.xlsx` record row and — via `ExcelStore._delete_unreferenced_photo_file` — the workspace `照片/` archive copy *if no other record references it*; the user's original imported file is never touched. Both single and batch unlink show a confirmation dialog stating originals are not deleted.
- Photo management is reachable from three right-click menus, all reusing the same handlers (`_replace_current_photo`/`delete_photo_at`/`_delete_selected_photos`/`_assign_voucher_to_selected`/`open_*`/`copy_*`): the `photo_table` menu (`_photo_table_context_menu`), the grid-cell menu (`_show_grid_context_menu`), and the single-preview menu (`_show_single_preview_context_menu`, via `PhotoGraphicsView.context_requested`). Grid cells support **Ctrl/Shift multi-select** (`_grid_selected_indices`); `_get_selected_photo_indices()` returns that set in grid mode, so move/批量删除 act on all selected. Grid multi-select resets to the current photo on every `_render_grid` (page/voucher/grid-count change).
- Fun cursors: the Settings dialog "光标样式" combo writes `cursor_style` (settings.json). Module-level `apply_app_cursor(style)` in `ui.py` deliberately uses per-top-level-window `setCursor()` — **not `QApplication.setOverrideCursor()`**, which would clobber I-beam/resize/busy cursors. Applied in `run_app()`, at the end of each `SpecimenWindow.__init__`, on settings save, and on restore-defaults.
- App icon variant: the Settings dialog "应用图标" combo writes `app_icon_variant` (settings.json). `apply_app_icon(variant)` in `ui.py` sets the icon on the `QApplication` + all top-level windows; applied in `run_app()` and on settings save/restore. `build_release.py` `--add-data`-bundles `assets/icons/app-icon-variants/` so the frozen app can switch icons at runtime; the exe file icon defaults to `DEFAULT_APP_ICON_VARIANT` (overridable with `--icon`).
- The build system uses PyInstaller's `--onedir` mode (fast start, no temp extraction) — not `--onefile`.
- `CURRENT_DATA_SCHEMA_VERSION` in `models.py` controls automated upgrades on workspace open: missing optional classification columns (`属名`, `目`, `纲`, `门`, `备注`) are appended to existing workbooks; hash-prefixed photo archives (`abcdef__original.jpg`) are migrated to clean names.
- **Schema version rollback**: `_assert_supported_data_schema()` raises `ImportConflictError` (hard-blocking) when the workspace's recorded schema version exceeds what the running software supports. The error message (v0.5.0+) tells the user to use "工具 → 降低工作区兼容版本…" in a newer version first. That action calls `store.downgrade_schema_version("1.0.0")` which resets the flag without touching data — since the schema migrations (classification columns, photo rename) are backward-compatible, the old software can read the data fine after the flag is lowered.

## Release & in-app update

- `build_release.py` packages each build into: the full per-platform zip (`setup_v{version}_{plat}.zip`, kept for backward compat / fallback; old name had Chinese prefix which GitHub strips on upload), a small `app_v{version}_{plat}.zip` (app code), a content-hash-named `runtime_{plat}_{hash}.zip` (the `_internal/` runtime), each with a `{zip}.sha256`, plus `update_manifest_{plat}.json`. `partition_bundle()` defines the app/runtime split (app = root exe ∪ `_internal/specimen_app/**` ∪ `.update_meta.json`). Each built bundle also gets a `.update_meta.json` recording its `runtime_hash` + `app_files`.
- `.github/workflows/release.yml` builds on `windows-latest` + `ubuntu-latest` when a `v*` tag is pushed, and uploads all zips + `.sha256` + `update_manifest_*.json` as GitHub Release assets.
- The app's **版本管理 → 软件版本 → 检查 GitHub 更新** button (and an opt-in startup check) uses `updater.py` to find a newer release and **incrementally** download it (reusing the runtime from a locally-installed version when `runtime_hash` matches — only the small app zip is downloaded), sha256-verifying everything and extracting into a complete, independent `releases/v{version}/`. New versions never overwrite old ones and are not auto-launched — the user switches manually, and `_launch_release()` offers a data snapshot first. See `docs/release-and-update.md`.
