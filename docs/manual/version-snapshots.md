# 数据版本快照

工具栏 / 工具菜单 → **版本管理**。

![版本管理](TODO_screenshots/version-snapshots.png)

## 创建快照

在版本管理对话框点 **创建数据快照** → 输入备注 → 把当前 `数据/` 全部文件复制一份到 `数据版本/v_YYYYMMDD_HHMMSS_备注/`。

## 还原快照

选中历史快照 → **还原**。还原前会**强制提示**先创建一个临时快照（防止误还原丢失最新数据）。

## 自动触发场景

下列操作会自动 / 主动询问创建快照：
- 合并 / 收件箱聚合前（强制）
- 软件版本切换前（询问）
- 工具 → 用 Excel 打开数据文件（提示）

## 兼容版本降级

工具菜单 → **降低工作区兼容版本**（密码门控 `ADMIN_PASSWORD`）→ 把 `工作区配置.json` 的 `data_schema_version` 写回 `1.0.0`，让**旧版软件**也能打开新版升级过的工作区。

降级**不动数据内容**，因为已发布的 schema 升级（分类列追加、照片重命名）都向后兼容。

## 检查更新

版本管理对话框底部 **检查 GitHub 更新** 或 **菜单栏 → 帮助 → 检查更新** —— 详见 [软件更新](update.md)。
