海洋动物标本图像机器学习物种分类综述与训练方案（2024-2026）

面向整体照、头部、疣足、刚毛等多部位图像，梳理当前机器学习用于物种分类的主流方法，并给出适合专业分类学家的专家参与式训练与软件集成路线。

1. 结论摘要

最优先测试BioCLIP 2、DINOv2、Bio-DINO 的 embedding 和 kNN 检索。当前标签不足，不宜直接训练大型监督模型。
最合理训练单位不是单张图片，而是一个标本个体及其多张证据图：整体、头部、疣足、刚毛、显微图。
关键短板当前已有 160 张 TIFF 原图，但 Excel 中专家确认物种标签极少。标签质量比模型选择更重要。
软件目标点击图片后给出 Top-k 分类候选、相似历史标本和自动填表建议，由专家确认后写入正式字段。

3. 2024-2026 研究进展

3.1 生物视觉基础模型

BioCLIP（CVPR 2024）是近年最重要的生物视觉基础模型之一，面向 Tree of Life 训练，适合生物图像的 zero-shot、few-shot 和 embedding 检索。

BioCLIP 2（2025）扩大到 TreeOfLife-200M，模型说明中提到约 214M 生物图像，强调层级对比学习和生物学 embedding 结构。对本项目应作为第一优先级。

Bio-DINO 是生物多样性图像自监督 encoder，训练于约 31M 生物图像，适合做纯视觉相似检索。

Insect-Foundation（IJCV 2025）说明类群专用视觉-语言基础模型正在成为趋势。

3.2 自监督表征

DINOv2（TMLR 2024）是强通用自监督视觉模型。对标签少的生物图像，常见路线是 frozen encoder + kNN / logistic regression / clustering。

3.3 细粒度视觉分类

FGVC 研究关注低类间差异和高类内差异。FET-FGVC（Pattern Recognition 2024）和 MCM-ViT（2025）都强调局部判别区域、多尺度和 mask 引导。对头部、疣足、刚毛等诊断性状非常相关。

3.4 LifeCLEF 与真实生物识别

LifeCLEF 2025 覆盖 AnimalCLEF、BirdCLEF+、FungiCLEF、GeoLifeCLEF、PlantCLEF。研究趋势是长尾、少样本、开放集、多标签、专家验证。

3.5 层级与开放世界

TaxonomyNet（Scientific Reports 2026）强调多阶元输出的一致性。对分类学软件而言，模型应同时输出科、属、种，并在种级不确定时退回到属/科级，而不是强制预测。

5. 主流方法体系

迁移学习分类EfficientNet、ConvNeXt、MobileNet、Swin、ViT。标签足够后训练稳定，但容易学到背景伪特征。
生物基础模型BioCLIP 2、BioCLIP、OpenCLIP。适合 zero-shot、few-shot、拉丁名候选和 embedding 检索。
自监督 embeddingDINOv2、DINOv3、Bio-DINO。适合少标签、聚类、相似标本检索。
局部/部位 FGVCattention、mask、part-based、multi-scale。适合头部、疣足、刚毛等诊断结构。
检测/分割 + 分类SAM、GroundingDINO、YOLO。用于去除黑底、标尺、色卡，裁剪主体或局部结构。
多图融合一个标本多张图，使用加权平均、attention pooling 或 MIL 输出标本级候选。

7. 详细训练方案

阶段 0：数据治理

  1. 扫描全部 TIFF，生成 `manifest.csv`。
  2. 计算 SHA256，建立原图与派生图映射。
  3. 读取 Excel 的照片、标本、分类信息。
  4. 输出缺失报告：有图无标签、有记录无文件、有分类无照片、疑似重复。

阶段 1：图像预处理

dataset_derivatives/
  jpg_1024/
  jpg_768/
  jpg_512/
  thumb_256/

原图保留不动，训练读取 JPEG/WebP 派生图。

阶段 2：embedding 与相似检索

先跑 BioCLIP 2、DINOv2、Bio-DINO，提取 embedding，做相似图检索和聚类。这一步最适合当前标签少的阶段。

阶段 3：zero-shot / few-shot

用拉丁种名、中文种名、属、科、形态短语构建候选词，测试 BioCLIP 2 输出 Top-k 的能力。

阶段 4：监督分类基线

标签足够后，用 EfficientNet-B0、ConvNeXt-Tiny、MobileNetV3、Swin-Tiny 训练。必须按 specimen_id 分组切分。

image_size: 384 或 512
batch_size: 4-16
amp: true
optimizer: AdamW
epochs: 30-100
early_stopping: true
class_balanced_sampler: true

阶段 5：部位识别器

先训练 `image -> view_type`，让软件能自动判断整体、头部、疣足、刚毛、显微图。

阶段 6：部位专用模型

分别训练 whole、head、parapodium、chaetae 的 embedding 或分类头。不同部位的解剖信息不同，不应全部混训。

阶段 7：标本级多图融合

score(taxon) =
  1.0 * mean(whole_scores) +
  1.5 * mean(head_scores) +
  2.0 * mean(parapodium_scores) +
  2.5 * mean(chaetae_scores)

后续可升级为 attention pooling 或 multiple instance learning。

8. 软件集成方案

关键规则 AI 不直接覆盖专家字段。AI 只写建议字段，专家确认后再写入正式分类信息。

点击图片后，AI 面板建议显示：

  • 当前图片部位：whole / head / parapodium / chaetae
  • 单图候选 Top-5
  • 标本级候选 Top-5
  • 相似历史标本 Top-10
  • 证据分解：整体、头部、疣足、刚毛分别支持什么
  • 操作：填入分类信息、只填科/属、加入待复核、驳回建议

推荐架构：

PyQt 主程序
  -> 本地 AI 推理服务 FastAPI/CLI
      -> 模型加载
      -> embedding 缓存
      -> top-k 预测
      -> 返回 JSON
